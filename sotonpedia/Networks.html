[edit] Networks

  • Aims
    • Electrical reliability: Avoid packet corruption
    • Handle congestion: Avoid throwing away packets due to congestion

[edit] MIMD

  • MIMD: Multiple instruction multiple data, with independent processors and distributed memory and a communications network processor to processor
  • Distributed Memory Machine: Each process can communicate with any other process but can only access its own local memory
    • Other Models: Data parallel (message passing: single process performing calculations over data structures in parallel), shared memory (multiple processors that share the same memory map)

[edit] Topologies

  • Concepts
    • Diameter (d): The worst case latency (worst case number of hops it has to go through)
      • The lonest path length between two processors
    • Valency/Node cost (v): Degreee of vertices (Number of links per node)
    • Value for money (d*v): Value for money

[edit] Full Connection

[edit] Complete Full Connection

There is a wire from every processor to every other processor.

  • Direct communication: To send a message to a processor, send down the appropriate wire to that processor
  • Properties: Each of P processors has P-1 bi-directional communication links, one for each of the other processors
  • Diameter: 1
  • Valency: P-1 (Number of links per node)
  • Value for money: P-1
  • Usage: Small shared/NUMA architectures, Message passing architectures
  • Evaluation
    • Expensive on link hardware: P-1 bi-directional links per processing node
    • Complex wiring: (P(P-1)/2) bi-directional links must be routed (divide by 2 as bi-directional)
    • Difficult to expand: Each processing node needs to be changed to add just one extra processing node
    • Reliability problems: Lots of connectors may break

[edit] Bus Connection

All processors share a single bus and compete for its use

  • Evaluation
    • Simple: Plug in processors, plug in memory and it works
    • Poor scalability: As you add processors, the shared memory bus gets too slow and becomes a bottleneck (limit of ~16 processors)
    • Poor performance
    • Low bandwidth: Bandwidth is shared between processing nodes and doesn't increase as more processors are added
    • Arbitration: Determening who can use the bus at a given time

[edit] Ring

Each processor connected to two other processors to form a ring with a fixed number of interconnects

  • Diameter: P/2 (One point of the ring to the opposite point of the ring)
  • Valency: 2
  • Value for money: P
  • Bisection width: 2 (Across the ring)
  • Evaluation
    • Distance and connection grows rapidly

[edit] Cube/Hypercube

Each prociessor connected in a cube formation, with dimensions.

  • Hypercube: Greater than 3 dimensions is a hypercube. Each processor connected to another in each dimension.
  • Usage: General purpose processing
  • Diameter: - same as the value of (Worst case is furthest dimension)
  • Valency: - same as the value of (Number of links as the dimensions)
  • Value for money: - same as the value of
  • Number of dimensions:
  • Bisection Width: (Removing a dimension)
  • Evaluation
    • Efficient routing

    • Average message distance grows as hypercube grows, but not too bad
    • Grows logarithmically with number of processors
    • More complex wiriting
    • Limitations: Number of nodes must be power of 2 (2^n nodes for dimension n)

[edit] Fat Tree

The links in a fat-tree become fatter towards the top of the tree from the root

  • Usage: Very common strategy for modern machines, general purpose application
  • Components
    • Processor nodes: Dedicated processor nodes with 1 port
    • Switch nodes: 8/32 ports or similar, connect local nodes to ports and connect to higher level switch
  • Fat: The communication links at the levels close to the root are duplicated/higher bandwidth
    • Lower levels: Local communications, less fat links
    • Higher levels: Increased traffic across levels, higher fat links
  • Scalability: Bisection bandwidth increases in propirtion to the size of the network
  • Binary fat tree:
    • Diameter:
    • Bisection width: (The number of nodes at the next level down on both sides)
  • Quarternary fat tree:
    • Diameter:
    • Bisection width: (The number of nodes at the next level down on both sides)
  • Bisection width: 1
  • Evaluation
    • Can be tailored to efficiently use any bandwidth available by chooisng the fatness of the links
    • The best networking strategy
    • Lack some of the advantages for particular parallel applications compared to partial (e.g. grid/cube). Rarely seen in large-scale parallel machines
      • Low connectivity per node compared to partial connected (e.g. grid, hypercube)
    • Good scalability: Scale well to a large number of nodes
    • Easy to upgrade: Can start off with a smaller switch at the top
    • Routing: Routing of messages takes place at the switches not the processing nodes

[edit] Performance

[edit] Message Latency

The time taken for the delivery of a message

  • Message latency:
    • n: number of hops
    • l: number of data bits
    • h: number of header bits
    • B: link bandwidth in bits per second
    • o: overhead for software routing

[edit] Bandwidth

  • Total bandwidth: Measure of best case, ignoring arrangement of links
  • Bisection bandwidth: The minimum volume of communication allowed between any two halves of the network
    • Usage: To work out how well you can move information globally accross the network
    • Bisection width: The number of edges to be removed to seperate the graph into two equal parts
    • Calculating: Product of the bisection width and the channel bandwidth
    • Division: Network divided in two in such a way as to give the worst possible bandwidth
    • Close to worst case
  • Bisection Widths
    • Completely connected:
    • Ring: 2 (Across the middle of ring)
    • Grid: (Accross the grid)
    • Cube/Hypercube: (Removing a dimension)
    • Fat tree: 1 (Breaking the root node in half)

[edit] Circuit

[edit] Packet

Messages are divided into packets before transmission. Each packet has a destination address and a fixed maximum length. They are re-assembled into the original message on arrival

[edit] Store and Forward

Information is sent to an intermediate station where it is kept and sent at a later time to the final destination or another intermediate station

  • How it works: Node absorbs the incoming message into memory and decides where to route it. If the outgoing link is busy, it remains in memory, otherwise it is sent out to the next link.
  • Performance: Assuming an empty network and no decision-making overhead, the performance is the time it takes is the time it takes to transfer accross each hop
    • Formula:
    • n: Number of hops
    • l: Number of data bits
    • h: Number of header bits
    • B: Link bandwidth in bits/second
  • Evaluation
    • Efficient: All links are kept free unless a message is being transferred
    • Slows down nodes: Processing at the nodes is slower as they have to absorb the whole message first
    • Have to wait: Have to wait for it to be received and forwarded at each node
    • Careful blocking: A single waiting packet can only block one node

[edit] Wormhole Routing

The head of the packet advanced directly from ingoing to outgoing channels on the routing chip. Packets are divided into flits (flow control digits). When a node examines the header flits, it selects the next channel on the route and begins forwarding flits down that channel. The header advances along the specified route and the remaining flits follow in a pipeline fashion.

  • Header flits: Govern the route
  • Properties
    • Small buffers: Each router only has a small buffer able to hold a small number of flits, not the whole message
  • Performance:
  • Evaluation
    • Pipeline fashion: Each node sending incoming flits on to the next node without waiting for the rest of the message
    • Better throughput - No waiting, the first flit does not wait for the rest of the message
    • Simpler: Can be implemented with small, fast switches
    • Stalling: When a link is blocked, everything behind it stalls as the block moves backwards and nothing can move forwards
    • Contention and Higher message latency: When a message is delayed due to a temporary blockage, it remains spread out accross the network, potentially blocking a lareg number of other messages
    • Less logic: Routers only need to read the header flits to determine the route

[edit] Virtual Cut-Through Packet Routing

Performs wormhole like routing where packets spread accross the network until a blockage, at which point the entire message buffers at the blocked node (line in Store-and-forward)

[edit] Switching

[edit] Circuit-Switching

A circuit switching network is one that establishes a circuit (or channel) between nodes and terminals before the users may communicate, as if the nodes were physically connected with an electrical circuit.

  • Usage: Where the switch cannot easily read content

[edit] Cross-bar switch

Capable of channeling data directly between any two devices attached to it up to its maximum number of ports

  • Evaluation:
    • Not easy to build: Require lots of chips and wire
    • Out of control: Can get out of control very easily

  • Middle layer: Bigger than the outer layers, to ensure spare links

    • m: The number of links incoming/outgoing from the middle layer
  • Rearrangebly non-blocking: Rearrangeably non-blocking when k = n
