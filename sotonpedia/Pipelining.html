[edit] Pipeline

Pipelining: A pipeline is a set of data processing elements connected in series, so that the output of one element is the input of the next one.

  • Advantages: Executing instructions like an assembly line - increases CPU instruction throughput (number of instructions completed per unit of time)
  • Parallelism:
    • Data level: All nodes run the same code but have different regions of the data
    • Algorithmic level: Different nodes run different parts of the code and the data flows past them
  • Disadvantantages: The problems arising from dependence, when the pipeline breaks (e.g. branches), increases the general overhead of execution of each instruction, it doesn't reduce execution time of individual instructions
  • Linear pipeline: Cascade of processing stages, linearly connected to perform a fixed function over a stream data flowing from one end to the other
  • Asynchronous: Data flow between adjacent stages is controlled through handshaking (e.g. send signals ready to transmit etc.)
  • Synchronous
    • Pipeline stages: Combinational logic circuits
    • Interfaces: Clocked latches between stages which isolate inputs from outputs - take out, do what you want, and put it back in
    • Equal delays in all stages
    • Controlled by clock pulse: All latches transfer data to the next stage simultaneously
      • Clock period: Based on the delays of the components - controls the speed of the pipeline. Set to be greater than the longest delay between pipeline stages
    • How it works: Instructions enter at one end, progress through the stages and exit at the other end
  • Multiple pipelines: Many modern processors have multiple instruction decoders each with its own pipeline allowing for multiple instruction streams - more than 1 instruction per clock

[edit] Properties

  • Different parts in parallel: Different parts of different instructions in parallel
  • Pipeline stage: Step in the pipeline
  • Pipe: Connection of pipeline stages together
  • Throughput: How many instructions exit the pipeline in a given time period
  • Processor cycle: Generally 1 (or in some cases 2) clock cycles

[edit] Considerations

  • Goal: Balance the time for each step in the process

[edit] Latches

  • Transparent latch: One bit of memory which is updated when an enable signal is raised
  • Wants
    • High speed (minimum time reqd to accept an input/save it)
    • Consistency of timing (making change to follow new input value the same - otherwise get skew)
  • Earle latch
    • One level of AND-OR logic can be built into the basic latch with no impact on switching characteristics
    • Introduces no additional gate delays
  • Timing
    • The problem: If had one clock line, then all latches transparent at the same time and data moves along the entire pipeline (not wanted - it'll all move along without being processed!)
    • Alternate clock lines: Different clock lines to different latches (two clock lines, connected to every other latch) - alternate clock lines on different edges (one down, one up)
  • Advantages
    • More tolerant: More tolerant of bad hardware
    • Can be used with non-overlapping clocks: When dealing with dodgy hardware, got different clocks, can't make d-types work
    • Strips out more logic
  • Problems
    • Complexity of using latches
    • Timing issues: Need to leave gaps big enough - Less time to do work because of synchronisation between different clock lines
    • Less gap: Propogation delay is just the gap between clocks
    • Slower: 1/3 of the speed - making half of them do something, then making the other half do something

[edit] Types

  • Arithmethic pipelining
  • Instruction pipelining: Streams of instructions
  • Processor pipelining
  • Static pipeline: Only one functional configuration
  • Dynamic pipeline: Permits several functional configurations to exist simultaneously

[edit] Usage

  • Instruction pipeline: Fetch, Decode (and read registers), Execute, Memory access (read/store), Write-back (write result to registers)
  • Mathematical pipelines: Mathematical operations pipe-lined (e.g. series of multiples, multiply/accumulates etc.)
    • e.g. Multiply, Division etc.

[edit] Issues

  • Clock skew
  • Pipeline hazards
    • Structural hazards: Resource conflicts where the hardware cannot support the combination of instructions
    • Data hazards: Data dependencies, depending on results from previous instructions
    • Control hazards: Pipe-lining branches/other instructions that change the PC
  • Problems
    • Exceptions
    • Stopping/restarting execution
