[edit] Buses

Buses are shared communication links used to connect muiltiple subsystems.

  • Considerations
    • Performance
    • Usability

[edit] Types

  • Width
    • Parallel
    • Serial
  • Usage
    • Process-Memory bus
      • Extremely important in modern PCs, because it is often a main bottleneck to system performance
      • Short and high speed
      • Only need to match memory system: maximise memory to processor bandwidth
      • Connects directly to processor
      • Optimised for cache block transfers
    • IO bus
      • I/O buses connect the CPU to all other components, except RAM.
      • Data are moved on the buses from one component to another, and data from other components to the CPU and RAM
      • Usually lengthier
      • Speed will always be lower than the system bus speed
      • Connects to processor-memory bus or backplane bus
    • Backplane bus
      • Backplane is an interconnection structure within the chasis
      • Allow processors, memory and I/O devices to coexist
      • Cost advantage: one bus for all components

[edit] Components

  • Organisation
    • Control lines: Signal requests and acknowledgements, indicate type of information on data line
    • Data lines: carry information between source and destination, Data and address, complex commands
  • Master and Slaves
    • Master: Issues commands and controls data transfer
    • Slave: One who responds

[edit] Characteristics

  • Different lengths
  • Many types of devices connected
  • Wide range in bandwidth
  • Bus standards: Published public interfaces (host and plug-in card do not need to know about each others implementation)

[edit] Design Decisions

  • Bus width: Seperating address and data lines
    • Performance: Seperate address/data lines
    • Cheap: Multiplex address/data lines
  • Data width
    • Performance: Wider is faster (e.g. 64 bits)
    • Cheap: Narrower is cheaper (e.g. 8 bits)
  • Transfer size: Simplicity of transfer
    • Performance: Transferring multiple words to have less overhead
    • Cheap: Single-word transfer is simpler
  • Bus masters and split transactions: How many bus masters (devices which can initate a transaction, e.g. the CPU)
    • Performance: Multiple bus masters (e.g. multiple CPUs, IO devices which can start transactions). Arbitration
      • Packet-switched buses: Using packets, can do split transactions rather than one thing holding the bus for a whole transaction
        • Dividing into requests and replies. Bus can be used between requests and replies.
        • Bus available to other masters while one master doing something
    • Cheap: Single master. No arbitration. No split transactions
  • Clocking
    • Performance: Synchronous - clock in the control lines, fixed protocol for sending address/data relative to clock.
      • Advantages: Little/no logic, faster as avoids overheads
      • Problems: Cannot be long due to skew, everything must run at the same clock rate
      • Better: Short and similar
    • Cheap: Asynchronous - not clocked - self-timed handshaking protocols used
      • Advantages: Accomdoate variety of devices with different clock rates, can widen bus without skew/sync problems
      • Problems: More overheads (syncing bus for each transaction), lower performance
      • Better: Varied and long
    • Implications: Data bandwidth, physical distance between devices and number of devices
  • Termination

[edit] Transactions

  • Read transaction: Transfers data from memory
    • Send address down bus to memory
    • Send read control (e.g. assert READ signal on the read line)
    • Memory responds by returning the data on the bus with appropriate control signals (e.g. deasserting the WAIT signal)
    • The read begins when the NOT READ signal is asserted (the control signal indicating a read should take place)
    • The data is available when the WAIT signal is deasserted
  • Write transaction: Writes data to memory
    • CPU/IO device sends address and data

[edit] Impedence

[edit] Skew

Skew is the differences in propagation delay along each data line may occur and not all bits may arrive at the destination at the same time.

  • Clock skew: The clock is out of sync with the data as the clock is on a seperate line
    • Relationship: When the bus speeds are doubled, the working distance of the bus is typically cut in half
  • Avoiding
    • Clock and data together: Encoding the clock and data on the same wire so they can't be out of step
    • One clock to a few data lines: 1 clock for every 4 data or so, easier to keep the same
    • Clock unskewing: Selective delays at receiving end to move the clock into the right place

[edit] Jitter

Jitter is the time variation of a periodic signal in electronics, often in relation to a reference clock source

  • Clock jitter: Instead of the ideal clock signal with a frequency and duty cycle (edges accur at exactly the same time each cycle), variations in timing of successive edges occurs, with some cycles being longer and others shorter.
    • Cycle-to-cycle variance

[edit] Dispersion

Dispersion is the problem where different frequencies travel through at different speeds, caausing to turn up out of shape.

  • Damaged edges: The edges (e.g. of a square wave) are damaged
  • Phase: High frequency may be out of phase by the time the data gets to the end of the transmission line
  • Wrong clock: If the clock is twice the frequency of the data, the clock will turn up at the wrong time

[edit] Power usage

  • Large signals: Signals must be large enough to overcome noise and poor termination (e.g. left over echoes from settling)

[edit] Evaluation

  • Advantages
    • Low cost: Single sets of shared wires
    • Simple: Electrically very simple
    • Versatility: New devices easily added, peripherals can be moved between systems
    • Abstraction: Manage complexity by partitioning the design
    • Interfaces: Abstracts complexity from host and plug-in cards
    • Usability: Self-configuring, auto-setup buses
  • Disadvantages
    • Communication bottleneck: Bandwidth of bus can limit make IO throughput
      • Uncontrolled electrical environment
      • Bouncing: Signals do not just flow down bus, arrive at destination and disappear. As buses aren't properly terminated, they bounce backwards and forwards
      • Bus settling time: The time it takes for a signal to go up and down a few times before it ceases to interfere with other signals
    • Doesn't scale well: As you add more devices, slowing it down - adding more capitance
      • Lowers bus impedence: You need more current to the bus
      • Increases bus settling time: Bus settling time gets longer
      • Speed differences: Slower old devices versus newer fast devices - have to speed whole bus down to slowest device
    • Length affects performance: Bandwidth controlled by size of bus and settling time
    • Phyiscal limitations: Maximum bus speed limited by Legnth, Number of devices
    • Handling varying devices: Widely different latencies and varying data rates
    • Slow: Slow compared to point-to-point systems
    • Difficult to have wide buses: Buses grow in complexity as they grow in size
    • Noise: Pick up noise on bus circuit
      • Example: PCIX - never caught on or performed - couldn't control the electrical issues accross too many bus
