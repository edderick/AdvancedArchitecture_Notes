[edit] CISC

[edit] RISC

  • Register to register
  • 3 arguments: Most instructions can take 3 arguments, with a destination register - plenty of registers to spare
  • Simple load and store

[edit] Vector

  • Single instruction to perform vector on vector operations
  • Example: Multiplying one vector of 100 with another vector of 100
  • Had large vectors of registers linked together with lots of wire
  • Doesnt help with things like matrices (rows against columns)

[edit] Superscalar

A superscalar CPU architecture implements a form of parallelism called instruction-level parallelism within a single processor. It therefore allows faster CPU throughput than would otherwise be possible at a given clock rate.

  • Execution: A superscalar processor executes more than one instruction during a clock cycle by simultaneously dispatching multiple instructions to redundant functional units on the processor
  • Redundant functional units: Can execute different instructions that use different units in parallel
  • Instruction level parallelism: Issuing multiple instructions per clock
  • Dynamic: Hardware dynamically combines instructions such that dependencies are obeyed
  • Data dependency graphs
  • Backwards compatible
  • High hardware complexity
  • Branch penalty in terms of instructions increases as machine parallelism increases

[edit] x86

  • Multiple data sizes and addressing methods
  • Limited number of registers
  • Variable length instructions

[edit] Dataflow

The dataflow model is one in which execution is driven only by the availability of operands

  • Avoids: The bottlenecks from the Von Neumann model
  • Model
    • Enabling rule: An instruction is enabled/executable if all operands are available
    • Firing rule: An instruction is fired/executed when it becomes enabled (consumption of its input data and generation of output data)
  • Usage: Digital signal processing, network routing, graphics processing, data warehousing
  • Dataflow languages: Single-assignment languages: A variable may appear on the left side of an assignment only once within the area of the program where its active
    • Examples: VAL, LUCID
  • Pure dataflow: Static, dynamic, uses token store
  • Hybrid dataflow: Augmenting dataflow with control structures


[edit] Influence

  • Didn't really take off in general purpose computing or large-scale parallelism
  • Out-of-order execution
    • Execution window: An execution window which follows the sequential order but instructions inside can be completed in data dependency order
    • Typically around 2-6 execution units and 32-200 instructions

[edit] VLIW

  • Static at compile-time: Compiler statically combines instructions such that data dependences are obeyed
  • Scheduling multiple instructions per clock at compile time inside VLIWs rather than at runtime
  • Medium hardware complexity

[edit] SIMD

Single instruction multiple data

  • Data level parallelism: Different processors performing the same operation on multiple data

[edit] MIMD

Multiple instruction multiple data

  • Seperate processors/computers that communicate with each other
  • Different processors executing different instructions on different data
  • Shared memory
  • NUMA: Non-uniform memory access
  • Non-shared memory
    • Message passing
  • Bus based or hierarchical


[edit] Latency Hiding

  • Interleaving execution of two+ instruction streams on the same harwdare to increase utilisation
  • MTA instructions: Three operations packed into 64-bit instruction words
    • Memory, arithmetic, arithmetic/branch
  • Run multiple independent execution threads on processor
  • Fine-grain multi-threaded

[edit] IA64

  • VLIW Instructions
    • 3 instruction slots of 41 bits
    • Template as to what instructions are
